% SUMMARY
% -------
% This paper proposes COREY, a system for entropy-guided operator fusion 
% in selective SSM (state space model) kernels, combined with Hadamard-based 
% feature reparameterization. The work spans theory (three theorems on entropy, 
% depth bounds, and quantization stability), a deterministic Python cost model 
% (Tier-1 prototype), and real-GPU feasibility checks (Tier-2) on Mamba checkpoints.
% 
% STRENGTHS
% -------
% 1. **Honest two-tier framing**: The paper clearly separates prototype 
%    diagnostics (Tier-1) from real-checkpoint feasibility (Tier-2), avoiding 
%    conflation of synthetic and hardware evidence.
% 2. **Real GPU results**: Table 3 (chunked selective-scan) demonstrates 
%    3.24× latency improvement on actual Triton kernels, closing a critical gap 
%    that plagued earlier drafts.
% 3. **Distribution-gap characterization**: The paper honestly reports that 
%    entropy *decreases* on real Mamba activations (0/80 pairs gain entropy),
%    contradicting the synthetic-regime theory. This is strength, not weakness.
% 4. **Structured formulation**: Theorems 1–3 provide formal grounding 
%    (doubly-stochastic mixing, depth bounds, quantization stability), elevating 
%    the narrative above heuristics.
% 5. **Comprehensive ablations**: Five prototype ablation axes (threshold, 
%    hyperparameter grid, tiling policy, precision, sequence length) show that 
%    entropy provides discriminative power independent of arithmetic intensity alone.

% MAJOR CONCERNS (Blocking / Near-Blocking)
% -------
% 1. **Chunked scan ≠ operator fusion (critical scope mismatch)**
%    - The main hardware result (Table 3, 3.24× speedup) measures chunk-size 
%      selection within a single operator (selective_scan_fn), NOT multi-operator 
%      fusion across heterogeneous SSM layers (in_proj, scan, elementwise, out_proj).
%    - This is correctly caveated in multiple places (caption, Section 6.4, 
%      Limitations 5, Conclusion) and the paper repeatedly says "end-to-end 
%      operator fusion across all SSM operator types remains future work."
%    - However, a reader skimming Title + Abstract + Table 3 might interpret 
%      the paper as claiming full-chain operator fusion, not single-operator 
%      kernel scheduling within selective_scan_fn.
%    - **Fix**: Title or Abstract could add phrase like "chunk-size selection" 
%      or "kernel-level scheduling" to clarify scope more prominently.
%
% 2. **Theorem 1 does not apply to real checkpoints (distribution gap)**
%    - Theorem 1 (entropy increase under doubly-stochastic mixing) holds 
%      empirically in synthetic heavy-tailed regime (34/35 instances, 
%      Sinkhorn L1 residual 0.070±0.010).
%    - On real Mamba-370M activations (80 layer-sample pairs), entropy 
%      *decreases* in all cases (mean gain −1.30 nats, L1 residual 1.689±0.037).
%    - The paper is honest about this, but the title and several sections imply 
%      the entropy-guided scheduling is a general principle, when in fact it is 
%      calibrated to and validated only on synthetic heavy-tailed distributions.
%    - The online hook that runs on real checkpoints (Table 2, hook-enabled runs) 
%      uses a *different* entropy signal (not the in_proj histogram that showed 
%      entropy decrease), so the hook's practical utility is disconnected from 
%      Theorem 1's theoretical justification.
%    - **Status**: This is well-disclosed in Theorem 1 Remark and Section 5.2.1, 
%      so not strictly a "major concern" but a key interpretive caveat that 
%      affects how much weight readers should place on the theoretical narrative.
%    - **Recommendation**: Reiterate in Conclusion that entropy scheduling is 
%      distribution-aware and may not generalize across all SSM subspaces.
