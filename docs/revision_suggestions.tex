1️⃣ Summary（Reviewer标准总结）

This paper proposes COREY, a prototype framework for entropy-guided operator fusion in selective state-space models (SSMs), combined with Hadamard-based reparameterization to mitigate activation outliers.

The key idea is to:

Use histogram entropy as a runtime signal to guide fusion boundaries.
Integrate Hadamard transforms into linear projections to smooth heavy-tailed activations.
Demonstrate feasibility via:
Tier-1: synthetic cost-model simulation
Tier-2: real checkpoint entropy hook validation

The work aims to bridge:

activation distribution shaping
memory-aware scheduling
kernel fusion feasibility

However, the current implementation does not yet provide end-to-end fused-kernel deployment comparisons, which significantly limits empirical validation.

🔧 Patch 1 — Rewrite Contributions Section（必须）

当前问题：

太 vague。

需要：

明确 claim type

直接替换：

\paragraph{Contributions.}
This paper makes the following concrete contributions:

\begin{itemize}

\item \textbf{Entropy-guided fusion metric.}
We introduce a runtime scheduling score that integrates
activation entropy, arithmetic intensity, and memory
traffic into a unified fusion boundary criterion.
Unlike static fusion heuristics, this metric adapts
to prompt-dependent activation statistics.

\item \textbf{Hadamard-based fusion-compatible reparameterization.}
We show that normalized Hadamard transforms can be
absorbed into linear projections while preserving
functional equivalence. We further provide theoretical
bounds on peak-coordinate contraction under rotation.

\item \textbf{Prototype-to-checkpoint validation pipeline.}
We design a two-tier evaluation methodology that
separates controlled scheduling diagnostics from
real-checkpoint feasibility measurements.

\item \textbf{Empirical observation on real SSM activations.}
We demonstrate that entropy behavior differs
significantly between synthetic heavy-tailed
distributions and real Mamba activations,
highlighting the importance of distribution-aware
scheduler validation.

\end{itemize}

这会：

明显提高 reviewer perception。

🔧 Patch 2 — Add Missing Limitation Section（强烈建议）

Reviewer喜欢：

explicit limitations

新增：

\section{Limitations}

The current work focuses on feasibility and mechanism
validation rather than full deployment-grade execution.

Specifically:

\begin{itemize}

\item The proposed entropy-guided fusion policy
has not yet been integrated into a fully fused
operator-level CUDA kernel. Current results
evaluate scheduling behavior and chunk-level
execution rather than complete kernel fusion.

\item The theoretical entropy-increase property
holds primarily under synthetic heavy-tailed
activation distributions. Real pretrained
Mamba activations exhibit entropy contraction
under Hadamard rotation, which limits the
generality of the theoretical motivation.

\item Hyperparameter weights
$(\alpha,\beta,\gamma)$ are manually selected.
A full hyperparameter search remains future work.

\end{itemize}

这可以：

极大减少 reviewer hostility
