\section{论文概述}
%% ============================================================

本文提出 COREY 框架，以激活熵作为运行时调度信号，用于选择性状态空间模型（SSM/Mamba）的算子
融合边界与 tile 大小决策，同时引入 Hadamard 重参数化以平滑重尾激活分布。评估分为两层：
Tier-1 基于 Python 代价模型的原型研究；Tier-2 在真实 GPU 上执行公开 Mamba checkpoint 的可行性
验证。

%% ============================================================
\section{主要优点}
%% ============================================================

\begin{enumerate}[label=\textcolor{strengthgreen}{\textbf{S\arabic*.}}]

  \item \textbf{问题定位清晰。}
    将熵增信号与 Hadamard 平滑、算子融合边界选择显式关联，形成有理论动机的统一框架，
    区别于现有纯启发式深度限制方案。

  \item \textbf{诚实的范围申明。}
    作者明确区分 Tier-1（代价模型）与 Tier-2（真实 GPU），并在 Limitations 节坦诚
    大量未完成工作，这种透明性在 systems 论文中值得认可。

  \item \textbf{理论框架基本完整。}
    三个定理（熵增、深度约束、量化稳定性）提供了方法的形式化基础，证明逻辑清晰，
    全文证明附于附录。

\end{enumerate}

%% ============================================================
\section{主要缺陷（致命问题）}
%% ============================================================

\subsection*{\textcolor{weaknessred}{W1（最严重）：核心评估缺失——无真实 GPU 方法对比基线}}

全文中 COREY（作为融合后端）从未在真实 checkpoint 上与 Static Fusion 或 No-Fusion
基线进行比较。Tier-2 仅展示熵 hook 可附加在推理循环上而不引入开销（Table~3），但 hook
本身并不执行融合。所有核心系统指标（延迟、吞吐量、DRAM bytes/token）均来自确定性
Python 代价模型（Tier-1），而该代价模型的验证仅依赖自身内部逻辑——构成循环论证。

\begin{notebox}
\textbf{建议：}在真实 GPU 上运行至少一个序列长度桶（如 $L=4096$）的
No-Fusion / Static Fusion / COREY 三路对比，并附 nsight 或 nvprof 剖析数据。
\end{notebox}

\subsection*{\textcolor{weaknessred}{W2：Theorem~1 的可适性条件实际上几乎无法验证}}

作者承认 Theorem~1 是有条件的（需要后旋转直方图可被双随机矩阵 $B$ 精确表示），
并使用 Sinkhorn 代理近似检验，$\ell_1$ 残差为 $0.070\pm0.010$。然而：
\begin{itemize}
  \item 35 个实例全部来自同一合成重尾生成器，分布覆盖极为狭窄；
  \item Sinkhorn 代理仅是 Birkhoff 多胞体上精确最小化子的近似，无法给出严格界；
  \item 定理本身对熵增益仅给出定性结论，与融合收益之间无定量连接。
\end{itemize}

实际上 Theorem~1 退化为``如果后旋转直方图可被混合，则熵增加''，对调度决策价值有限。

\begin{notebox}
\textbf{建议：}补充在真实 Mamba checkpoint 激活（非合成数据）上的 Sinkhorn 残差分析，
或将 Theorem~1 降格为 Proposition 并明确其局限性。
\end{notebox}

\subsection*{\textcolor{weaknessred}{W3：代价模型（Tier-1）的设计未经独立验证}}

Tier-1 的延迟数字来自``确定性 Python 代价函数''。Table~9 注释亦承认：近零方差（$\pm0.12$\,ms）
是代价模型本身的产物，而非 GPU 硬件行为。NeurIPS 审稿人通常不接受仅基于代价模型
的 speedup 数据作为主要证据。

\begin{notebox}
\textbf{建议：}将 Tier-1 数字整体移至附录，主文本仅保留 Tier-2 实测结果。
\end{notebox}

\subsection*{\textcolor{weaknessred}{W4：$(\alpha,\beta,\gamma)$ 超参数选取缺乏充分验证}}

消融实验（Table~7/11）仅覆盖粗粒度 $3\times3$ 网格（$\alpha\in\{0,0.45,0.90\}$）的部分配置，
且作者承认完整三路搜索是 future work。这三个超参数直接决定调度边界，
缺乏系统搜索使得``默认配置（0.45, 0.35, 0.20）接近最优''的隐含假设无法成立。

\subsection*{\textcolor{weaknessred}{W5：LongBench 分数极低且无法支撑任何质量声明}}

Table~4 中所有模型在 MultifieldQA-EN 上的 exact match 均为 0.000，NarrativeQA F1 低至
0.013--0.019。即使作为``harness correctness''验证，零分的 MF-EN 也令人怀疑
prompt 构建或解码配置是否正确，但作者对此未给出任何解释。

\section{次要问题}
%% ============================================================

\begin{enumerate}[label=\textcolor{noteblue}{\textbf{w\arabic*.}}]

  \item \textbf{Broken cross-references（附录）。}
    附录中存在多处 \texttt{Table ??} 与 \texttt{Section ??} 未解析引用，
    表明主文档与附录之间的 \texttt{\textbackslash label}/\texttt{\textbackslash ref}
    系统存在问题，需在提交前全部修复。

  \item \textbf{结构冗余。}
    Tier-1/Tier-2 区分说明在 Abstract、Introduction、Section~5、Section~6、Conclusion
    中重复出现至少五次，占用大量篇幅而信息增量极低，建议精简至一处并在其余位置仅作交叉引用。

  \item \textbf{RTX~3090 延迟异常未解释。}
    RTX~3090 上 Mamba-370M 平均延迟为 27205\,ms，而 RTX~3070 上仅 1384\,ms（Table~4）。
    作者归因于``存储路径开销''但未提供任何隔离实验或分析，
    严重影响读者对 Tier-2 整体可靠性的判断。

  \item \textbf{Table~3 中负延迟 delta 的来源未解释。}
    hook 附加后延迟反而降低（$-3.63\%$ 到 $-4.02\%$），
    1-sample/0-warmup 的测量噪声完全可以解释该现象，
    但作者未提示读者不应将其解读为真实加速效果。

  \item \textbf{Multi-GPU 结果（Section~6.6）与 COREY 方法无关。}
    数据并行分片只是在多 GPU 上独立运行相同基线推理，
    与熵引导融合调度无关，放在 Results 节中容易产生误导。

\end{enumerate}

\subsection{Fix 5：将 Theorem~1 的 applicability condition 升格为 Remark 环境}

首先，在 \texttt{main.tex} preamble 的 \verb|\newtheorem{theorem}{Theorem}| 之后添加：

\begin{verbatim}

\end{verbatim}

然后，将 \verb|\noindent\textbf{Applicability condition.}| 整段替换为：

\begin{verbatim}
\begin{remark}[Conditional applicability of Theorem~\ref{thm:entropy}]
\label{rem:applicability}
Theorem~\ref{thm:entropy} is conditional: it requires the post-rotation
histogram to be representable as a doubly-stochastic mixing of the
pre-rotation histogram over the \emph{same} fixed bins.  This condition
may fail when rotation redistributes probability mass across bin
boundaries in a manner not captured by any doubly-stochastic map on the
fixed grid.  We therefore use the theorem as a formal explanation of
\emph{when} Hadamard rotation increases histogram entropy, rather than
as an unconditional guarantee for every rotated activation distribution.
The Sinkhorn-proxy results below provide an empirical check on this
condition for synthetic heavy-tailed activations only; verification on
real checkpoint activations remains future work.
\end{remark}
\end{verbatim}

\subsection{Fix 7：Limitations 节重组——将设计局限与未完成工作分开}

将 \texttt{main.tex} 中 \verb|\section{Limitations}| 的整个 \texttt{enumerate}
替换为：

\begin{verbatim}
\section{Limitations}

\paragraph{Methodological limitations.}
\begin{enumerate}
  \item \textbf{Entropy coarseness.}
    Entropy is a coarse statistic and can miss fine-grained channel
    interactions; threshold selection may require per-model retuning.

  \item \textbf{Hardware resource saturation.}
    Deeper fusion is bounded by register and shared-memory limits;
    benefits may saturate on devices with restricted on-chip memory.

  \item \textbf{Activation distribution dependency.}
    Hadamard-based smoothing is most effective for heavy-tailed
    activations; models without pronounced outliers may see smaller gains.

  \item \textbf{Diagnostic proxy scope.}
    The low-bit diagnostic proxy is not a substitute for perplexity or
    downstream task accuracy, and is circular with respect to the
    entropy-based scheduling criterion it evaluates.
\end{enumerate}

\paragraph{Open experimental gaps (future work).}
\begin{enumerate}
  \setcounter{enumi}{4}
  \item \textbf{Narrow checkpoint coverage.}
    The LongBench subset covers only four tasks at 20 samples each with
    non-harmonized prompt caps across checkpoint scales; only one external
    baseline model scale has been run.

  \item \textbf{Absent method-versus-baseline runs.}
    Static Fusion and COREY have not been executed as real fused kernels
    on any checkpoint; the current Tier-2 evidence verifies the benchmark
    harness and deployment reference path, not the method itself.

  \item \textbf{Quantization access constraints.}
    Generic AWQ/GPTQ stacks remain unusable for Mamba.
    Quamba~\citep{chiang2024quamba} is installed but quantized-checkpoint
    inference and performance benchmarking remain unexecuted.

  \item \textbf{Single-sample hook benchmark.}
    The GPU-timed entropy-hook result (Table~\ref{tab:hook_micro}) is a
    one-sample, zero-warmup feasibility check, not a throughput measurement;
    the negative latency deltas are within measurement noise.
\end{enumerate}
\end{verbatim}

\section{给作者的总体建议}
%% ============================================================

本文的核心思路（熵作为融合调度信号 + Hadamard 重参数化）在概念上有价值，
理论框架基本完整。但当前提交的根本问题是：\textbf{方法本身从未在真实硬件上被验证}。
Tier-2 展示的是 hook 可以运行，而非 COREY 融合有效。

若要达到 NeurIPS systems 论文的基本门槛，至少需要：

\begin{enumerate}
  \item 在真实 GPU 上实现至少一个 Triton 融合算子，对
    No-Fusion / Static Fusion / COREY 进行 nsight-profiled 三路对比；
  \item 在真实 checkpoint 激活（而非合成数据）上验证 Sinkhorn 代理条件；
  \item 将代价模型延迟数字从主文本移至附录，不再作为主要证据；
  \item 解释或修复 MultifieldQA-EN exact match 全部为 0.000 的问题；
  \item 解释 RTX~3090 上 27205\,ms 与 RTX~3070 上 1384\,ms 的 20 倍差异。
\end{enumerate}

在完成以上工作之前，建议将本文定位为
\textbf{workshop paper}（如 EfficientML @ NeurIPS）而非主会投稿。
